SQL Client MCP — Cloud Infrastructure

AI chat agents are great at conversation but can't access real business data. There was no secure, production-grade bridge between AI assistants and data warehouses.

Built an online MCP (Model Context Protocol) server on AWS Lambda that implements the full Streamable HTTP spec — session management, tool discovery, and execution. AI agents send natural language, the server executes SQL against the data warehouse, and returns formatted results. DubGround — AI Video Dubbing

Dubbing a video into another language requires expensive voice actors and manual audio engineering per language. A 10-minute video can take days and thousands of dollars to dub.

Built an end-to-end pipeline: upload a video → detect speakers (pyannote) → transcribe (Omnilingual 7B) → contextual translation (Qwen 3.5 4B with overlapping-window batching) → clone each speaker's voice (Qwen3-TTS) → generate dubbed audio. Each speaker's vocal identity is preserved across languages. Alternative SeamlessM4T speech-to-speech path skips the pipeline entirely. TTS uses lazy-loaded models with LRU eviction on a single T4 GPU.

Reinventing Product Management

What if a PM could go from idea to working demo in hours, not sprints?

01

Requirements from Code + Data

Instead of writing requirements from scratch, the AI analyzes the existing codebase and real usage data to generate structured requirement documents — with acceptance criteria, edge cases, and technical constraints already filled in.

02

Automated Design & Task Breakdown

Requirements flow into design documents and implementation task lists. Each task is scoped, ordered by dependency, and ready for execution — no sprint planning meeting needed.

03

Working Demo from Spec

The AI executes each task against the actual codebase — writing production code, not prototypes. The PM reviews each step, and at the end has a working feature branch ready for engineering review.

04

Data Analysis on Demand

Need to validate an assumption? The analytics agent pulls real data mid-workflow. Need a benchmark for the PRD? It queries the warehouse and fills in the numbers. Data isn't a separate step — it's embedded in the process.

Applied this workflow to ship 9 feature specs for a production Android app — from account recovery flows to subscription metrics. Each spec went from requirement document to working code with tests.
