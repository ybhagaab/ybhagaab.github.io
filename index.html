SQL Client MCP Local

Published on npm-pretty-much

A local MCP server that provides a framework for solving the goldfish memory problem for AI coding assistants. Connects Kiro, Claude Desktop, and other tools directly to PostgreSQL and Redshift — with on-demand schema context presets that give agents persistent domain knowledge across conversations.

🗄️ PostgreSQL & Redshift 🔑 3 Auth Methods 📋 Schema Context Presets 🔒 SSL/TLS Modes 🚇 SSH Tunnel Support 8 SQL Tools

The framework introduces presets — bundled domain knowledge files (markdown or JSON) that agents load on demand before writing SQL. Teams encode their table docs, required filters, query patterns, and business rules into presets, and any AI assistant that connects instantly understands the domain. Supports direct credentials, IAM temporary credentials (auto-refreshed every 14 min), and AWS Secrets Manager for flexible auth.

DubGround

In Development

An end-to-end video dubbing platform that detects speakers, clones their voices, and generates multilingual dubbed audio — preserving each speaker's vocal identity across languages.

🎙️ Speaker Diarization 📝 Dual ASR Engines 🌍 Contextual Translation 🗣️ Voice Cloning 🔊 Text-to-Speech 🔄 Speech-to-Speech ⚡ Real-time Progress 🧪 Model A/B Testing

Video dubbing traditionally requires expensive voice actors and manual audio engineering per language. DubGround automates the full pipeline — from speaker detection through voice cloning to synthesis. Two workflows: the primary pipeline (diarization → ASR → contextual translation → voice cloning → synthesis) and an alternative SeamlessM4T speech-to-speech path that skips the pipeline entirely. The translation layer uses Qwen 3.5 4B with overlapping-window batching to preserve cross-segment coherence and dramatic fidelity. TTS uses lazy-loaded models with LRU eviction on a single T4 GPU — two model variants (~5.2GB each) swap in/out of 16GB VRAM on demand.

The Pipeline

Upload → Audio Extract → Diarization → ASR → Translation → Voice Clone → Dubbed Audio
